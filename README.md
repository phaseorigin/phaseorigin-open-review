# PhaseOrigin Open Review

This repository is a public issue tracker for bounded critical review of the CHC Framework Series.

It is not a request to accept or reject the whole CHC program at once. The useful unit of review here is a specific claim family, failure condition, missing reference, unclear boundary, reproducibility issue, or better formulation.

## Primary Entry Point

- Reviewer page: https://www.phaseorigin.com/research/review/
- Archive DOI: https://doi.org/10.5281/zenodo.20282162
- Open issues: https://github.com/phaseorigin/phaseorigin-open-review/issues

## Current Review Routes

Open route threads are organized around claim families:

- [Detector records](https://github.com/phaseorigin/phaseorigin-open-review/issues/1)
- [Classical objectivity](https://github.com/phaseorigin/phaseorigin-open-review/issues/2)
- [Interfacial branch budgets](https://github.com/phaseorigin/phaseorigin-open-review/issues/3)
- [Public-data stress windows](https://github.com/phaseorigin/phaseorigin-open-review/issues/4)

Each route keeps standard physics tools in scope and asks what, if anything, the CHC layer adds beyond ordinary language, existing theory, or retrospective labeling.

## Useful Issues

Useful issues usually include one or more of:

- a missing or better standard reference
- a case where a proposed distinction collapses into existing theory
- a boundary condition or assumption that is incorrect
- an example where a claim label is overstated
- a reproducibility, data, formalization, or notation problem
- a sharper formulation of the same criticism

Concrete criticism is preferred over broad agreement or disagreement.

New issues should use the review-objection template when possible. The preferred shape is: claim family, target page or paper, objection type, proposed status change, evidence or reference, and what would resolve the issue.

## Not Useful Here

Please avoid:

- promotional comments
- unbounded speculation
- general reactions to the whole framework without a target claim
- requests to replace standard physics with CHC language
- private or unpublished material that should not be posted publicly

## Local Issue Snapshot

The `issues/` directory is a local markdown snapshot of GitHub Issues. GitHub Issues are not included by a normal `git clone`, so this folder exists to keep the current public review routes inspectable from the repository itself.
